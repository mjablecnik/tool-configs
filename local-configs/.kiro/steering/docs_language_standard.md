# Documentation Language Standards

## English-Only Documentation

All technical documentation must be written in English to ensure accessibility for international developers and maintainers:

- README files
- API documentation
- Code comments and inline documentation
- Architecture documents
- Setup and installation guides
- Deployment instructions
- Troubleshooting guides
- Changelog and release notes
- Contributing guidelines
- Technical specifications
- Developer guides
- Steering files (like this one)

## Rationale

English documentation ensures:
- International developers can understand and contribute to the project
- Consistency with industry standards
- Better integration with global development tools and platforms
- Easier knowledge sharing and onboarding
- Professional presentation for open-source or collaborative projects

## Exceptions

- User-facing documentation intended for Czech end-users (e.g., user manuals, help pages on the website)
- Marketing materials and content targeting Czech audience
- Business requirements documents written for Czech stakeholders (if explicitly requested)

## Examples

✅ Good:
```markdown
# Installation Guide

Run the following command to install dependencies:
```

❌ Bad:
```markdown
# Průvodce instalací

Spusťte následující příkaz pro instalaci závislostí:
```

## Note

This rule applies to all technical documentation. User-facing content on the website should remain in Czech as per the target audience requirements.
